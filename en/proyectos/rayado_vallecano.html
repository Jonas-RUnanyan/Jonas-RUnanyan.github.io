Project Summary

What I learned from this project:

  • Worked in a multidisciplinary team using Scrum and Kanban
  • Designed spritesheets and simple 2D animations, as well as implemented them in Unity
  • Designed unit tests and conducted user testing
  • Composed several short and loopable tracks for the soundtrack
  • Documented my work in a GDD

Rayado Vallecano is the game we delivered as a project for the Video Game Fundamentals course in the Computer Engineering degree at UPM. In it, you play as Juan, a boy who just moved to Vallecas and must recover his new stolen phone without using violence while meeting his new neighbors.

This project was developed in 4 sprints, where team members rotated roles. In the first sprint, I set up what would become the sprite system.
In the second, we planned to use open-source sprites since none of us had design training, but seeing that we needed very specific assets, I tentatively created some sprites, like the trash containers and the garbage truck sprite.

In the third sprint, I was assigned to the QA team, where I performed unit tests on features developed by my teammates. Additionally, we conducted user satisfaction tests, incorporating the changes we deemed necessary. Finally, in the fourth sprint, the polishing stage, I created all the sprites for the Metro environment, including the train and its animation. I also composed a simple chiptune soundtrack based on recurring leitmotifs.
